# AI News Pipeline - 要件定義

## ゴール

> **毎朝、AI関連の最新ニュースを自動で収集・要約し、Slack通知とWebページで確認できるツール**
>
> ポートフォリオとして「LLM活用」「データパイプライン」「自動化」「クリーンコード」を総合的にアピール

## 利用シーン

- 毎朝のAIニュースキャッチアップ
- 週次/月次のトレンド振り返り

## 機能要件

| # | 機能 | 詳細 | 技術アピール |
|---|------|------|-------------|
| 1 | ニュース収集 | HN APIからAI関連記事を取得 | データパイプライン |
| 2 | 記事スクレイピング | URLから本文を取得 | データパイプライン |
| 3 | LLM要約 | 記事を日本語で要約 | LLM API活用 |
| 4 | Slack通知 | 毎朝の要約配信 | 自動化 |
| 5 | Webページ | 過去ニュースの閲覧・検索 | フルスタック |
| 6 | 自動実行 | 毎日定時に収集〜配信 | CI/CD |

## 技術スタック

| 領域 | 技術 | 理由 |
|------|------|------|
| 言語 | TypeScript | 型安全・保守性 |
| ランタイム | Node.js (ESM) | - |
| テスト | Vitest + Playwright | ユニット + E2E |
| LLM | OpenAI or Anthropic | 要約生成 |
| フロント | Next.js (SSG) | React + 静的出力 |
| ホスティング | GitHub Pages | 無料・簡単 |
| CI/CD | GitHub Actions | 定時実行・自動デプロイ |
| 通知 | Slack Webhook | シンプル |

## 品質要件（ポートフォリオ向け）

| 観点 | 内容 |
|------|------|
| 型安全 | strict mode、Zod等でバリデーション |
| テスト | カバレッジ80%以上目標 |
| ドキュメント | README、アーキテクチャ図、ADR |
| エラー処理 | リトライ、ログ、通知 |
| 設計 | クリーンアーキテクチャ意識 |

## 実行頻度

- 毎日1回（朝）定時実行
- GitHub Actionsのcron設定で自動化

## 今後の拡張候補

- ニュースソースの追加（Reddit、Twitter等）
- 週次ダイジェストメール
- トレンド分析・可視化
